IBM [watsonx Orchestrate](https://www.ibm.com/products/watsonx-orchestrate?utm_source=ibm_developer&utm_content=in_content_link&utm_id=tutorials_integrate-watsonx-orchestrate-agents-custom-ui&cm_sp=ibmdev-_-developer-tutorials-_-product) is an AI platform that helps you create, deploy, and manage autonomous agents to automate business processes. It offers a single platform for both developers and business users to quickly build agents. You can create no-code agents in an easy-to-use interface and view the results instantly. To learn more about watsonx Orchestrate and why developers use it, see [Get to know watsonx Orchestrate](https://developer.ibm.com/blogs/watsonx-orchestrate-june2025/) on IBM Developer.

Many organizations have already built chatbots, AI assistants, or conversational UIs with frameworks such as React or Vue. As the market evolves, agentic capabilities need to be added without replacing the existing front-end.

In this tutorial, learn how to integrate no-code or low-code AI agents that are built on IBM watsonx Orchestrate into any custom user interface.

## Architecture of integrating AI agents into a custom UI

This following figure explains how a user request in a custom chat interface is processed end-to-end, from the front-end UI to watsonx Orchestrate agents.

![arch](https://developer.ibm.com/developer/default/tutorials/integrate-watsonx-orchestrate-agents-custom-ui/images/arch.png)

The flow is as follows:

- A user sends a chat message through a React-based interface to request invoice analysis.
- [FastAPI](https://fastapi.tiangolo.com/), which acts as a wrapper, forwards the request to the watsonx Orchestrate Supervisor Agent.
- The Supervisor Agent interprets the request by using an LLM (llama-3.2-90B-Vision-Instruct from Meta) and routes it to the [Spend Analysis Agent](https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent/spendanalysis_nocode_agent) or the agent that you want to integrate.
- The [Spend Analysis Agent](https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent/spendanalysis_nocode_agent) or the agent that you want to integrate, retrieves data, processes it, and sends the response back through the Supervisor Agent.
- FastAPI formats the findings into a structured response, which is displayed in the custom chat interface.

## Prerequisites

For detailed instructions, see [Develop agents with no-code by using watsonx Orchestrate](https://developer.ibm.com/tutorials/develop-agents-no-code-watsonx-orchestrate/).
- A custom user interface built with React. Download the sample application from the [GitHub repository](https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent). It includes back-end and front-end code that is used in the tutorial.
- A REST client such as Postman or curl to test the APIs.
- An IBM Cloud account. You can also sign up for a [trial account](https://cloud.ibm.com/register?utm_source=ibm_developer&utm_content=in_content_link&utm_id=tutorials_integrate-watsonx-orchestrate-agents-custom-ui&cm_sp=ibmdev-_-developer-tutorials-_-trial).

## Steps

1. Understand and validate AI Agent APIs
2. Build a FastAPI wrapper
3. Deploy the FastAPI in IBM Code Engine
4. Integrate the APIs into a custom user interface

### Step 1. Understand and validate AI Agent APIs

The React [front-end code](https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent/frontend_code) works as a conversational interface. User instructions are sent to the AI agent in watsonx Orchestrate for processing. The agent identifies the intent, acts, and sends the response back to the front-end through APIs. In this step, explore the APIs needed to connect the AI agent with the interface.

1. **Generate token**: Start by authenticating with an API key. This returns a bearer token that you must include in all API calls.

1. Log in to your watsonx Orchestrate instance.
1. Go to **Profile → Settings → API Details**.
1. Click **Generate API Key**.
1. Copy the **Service Instance URL** and the API key. You will use them in the next steps.

   ![wxo_apikey_url](https://developer.ibm.com/developer/default/tutorials/integrate-watsonx-orchestrate-agents-custom-ui/images/wxo_apikey_url.png)

   Sample instance URL (referred to as **YOUR_INSTANCE_URL**):

   ```
   https://api.eu-central-1.dl.watson-orchestrate.ibm.com/instances/20250508-1435-1457-50e0-b8f069e11f66
   ```

   Copy codeCopied!

1. Use the following command to generate a token:

   ```
   curl --location 'https://iam.platform.saas.ibm.com/siusermgr/api/1.0/apikeys/token' \
    --header 'accept: application/json' \
    --header 'content-type: application/json' \
    --data '{"apikey": "YOUR_API_KEY"}'
   ```

   Copy codeCopied!

   ![generate_token](https://developer.ibm.com/developer/default/tutorials/integrate-watsonx-orchestrate-agents-custom-ui/images/wxo_generate_token.png)

   > **Note**: If you are using a watsonx Orchestrate instance on IBM Cloud, run the following command to generate the token:

   ```
   curl -X POST \
    "https://iam.cloud.ibm.com/identity/token" \
    --header "Content-Type: application/x-www-form-urlencoded" \
    --header "Accept: application/json" \
    --data 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=<YOUR_API_KEY>'
   ```

   Copy codeCopied!

1. **Get the AI Agent ID**: Multiple agents are deployed in the IBM watsonx Orchestrate platform. Each agent in watsonx Orchestrate has a unique `agent_id`. To find it, list all agents in your instance.

   For this tutorial, look for the ID that matches the [Spend Analysis Agent](https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent/spendanalysis_nocode_agent) or the agent that you want to integrate.

```
curl --location 'YOUR_INSTANCE_URL/v1/orchestrate/agents' \
    --header 'Authorization: Bearer <REDACTED_TOKEN>' \
    --header 'Accept: application/json'
```

Copy codeCopied!

![get_agent_id](https://developer.ibm.com/developer/default/tutorials/integrate-watsonx-orchestrate-agents-custom-ui/images/wxo_get_agent_id.png)

3. **Send a message to the agent**: Use this API to start or continue a conversation with the AI agent. Call it each time the user enters a message.
   - On the first `/runs` call, keep the `thread_id` blank.
   - On follow-up calls, include the `thread_id` from the response to keep the conversation in the same session.

     ```
     curl --location 'YOUR_INSTANCE_URL/v1/orchestrate/runs?stream=true&stream_timeout=120000&multiple_content=true' \
       --header 'Authorization: Bearer <REDACTED_TOKEN>' \
       --header 'IAM-API_KEY: <REDACTED_API_KEY>' \
       --header 'accept: application/json' \
       --header 'Content-Type: application/json' \
       --data '{
       "message": {
           "role": "user",
           "content": "show duplicate invoices"
       },
       "agent_id": "73c12457-0467-4204-9f68-30502f99e015",
       "thread_id": "f56c36f5-480a-4ff7-ad91-860787acc786"
       }'
     ```

     Copy codeCopied!

     ![start_conversation](https://developer.ibm.com/developer/default/tutorials/integrate-watsonx-orchestrate-agents-custom-ui/images/wxo_start_conversation.png)

     You now know how to generate a token, call the APIs, send a message to the agent, and display the response on the chat interface.

### Step 2. Build a FastAPI wrapper

In this step, create a FastAPI service that wraps the watsonx Orchestrate APIs. The service provides a single REST endpoint _/chat/v2_ to post questions and keeps the [user interface](https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent/frontend_code/) separate from server-side processing.

Next, deploy the FastAPI service on [IBM Code Engine](https://www.ibm.com/products/code-engine?utm_source=ibm_developer&utm_content=in_content_link&utm_id=tutorials_integrate-watsonx-orchestrate-agents-custom-ui&cm_sp=ibmdev-_-developer-tutorials-_-product), which standardizes the API contracts and runs in a fully managed, serverless environment with built-in scalability and efficiency.

1. Clone the [FastAPI service](https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent/backend_code/) to your local system:

```
git clone https://github.com/IBM/oic-i-agentic-ai-tutorials/tree/main/i-oic-integrate-headless-ai-agent/backend_code
```

Copy codeCopied!

2. Open the code in VS Code. The service ( _api/main.py_) is built with Python, but you can use Node.js or any framework of your choice.

3. Change the current directory to where the _Dockerfile_ is located:

```
cd oic-i-agentic-ai-tutorials/i-oic-integrate-headless-ai-agent/backend_code
```

Copy codeCopied!

4. Rename _env.sample_ to _.env_ in the _backend_code_ folder and update the values:

```
YOUR_INSTANCE_URL=your_instance_url
    API_KEY=your_api_key
    CLOUD_API_KEY=your_ibm_cloud_api_key
```

Copy codeCopied!

5. Install the required dependencies:

```
pip install -r requirements.txt
```

Copy codeCopied!

6. Run the app with Uvicorn. The entry file is **main.py** in the **api** folder:

```
uvicorn api.main:app --reload --port 8080
```

Copy codeCopied!

![backendcode_editor](https://developer.ibm.com/developer/default/tutorials/integrate-watsonx-orchestrate-agents-custom-ui/images/backendcode_editor.png)

7. FastAPI is now running locally at `http://localhost:8080/docs`.

## Summary and next steps

In this tutorial, you learned how to integrate a no-code AI agent built with the watsonx Orchestrate platform into a custom user interface. This shows how you can quickly add agentic capabilities to your applications by using simple API integrations.

IBM watsonx Orchestrate provides a flexible way to build and manage agents with no-code, low-code, or pro-code approaches. It also supports integration with external agents that are built on frameworks such as CrewAI, AutoGen, or LangGraph, making it a powerful platform for modernizing both new and existing applications.

As a next step, think about how agentic AI can help streamline workflows, improve productivity, and bring automation into your organization while still working within your current architecture and tools.
